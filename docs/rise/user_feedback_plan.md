# 사용자 피드백 수집 계획

이 문서는 실제 조사 결과가 아니라 실행 양식이다. 참여자의 동의 없이 건강정보나
직접 식별정보를 수집하지 않는다.

## 대상과 최소 규모

- 실제 참여자 총 5명 이상
- 고령자 1명 이상 필수
- 보호자 1명 이상 필수
- 나머지는 고령자·보호자·일반 주민으로 구성

## 공통 과업

1. 증상 관련 안전한 안내와 가까운 기관 찾기
2. 읍면동을 이용한 병원·약국 찾기
3. 기관 전화·주소·운영시간·출처 확인
4. 응급 또는 자살위기 문장의 즉시 연락 버튼 확인
5. 답변의 공식 출처 카드와 근거 식별정보 확인

## 익명 평가 항목

5점 척도로 글자 가독성, 질문 이해도, 답변 친절성, 출처 신뢰도, 기관 연결 편의,
안전 안내 명확성을 수집한다. 자유 의견은 개인·건강 식별정보를 제거한 뒤 분류한다.

## 결과표 필수 열

`participant_group`, `scenario_id`, `completed`, `time_seconds`,
`readability`, `clarity`, `trust`, `connection_ease`, `safety_clarity`,
`redacted_comment`, `action_taken`

완료 보고에는 대상별 참여 수, 과업 성공률, 평균 소요시간, 항목별 평균, 주요 개선
요청, 반영 커밋을 포함한다. 실제 데이터가 없으면 결과 수치를 작성하지 않는다.

수집 명령:

```powershell
& ".\.venv-p1\Scripts\python.exe" scripts\collect_rise_user_evaluation.py --collect-count 5
```

명령을 한 번 실행하면 실제 사용자 1/5부터 5/5까지 이어서 진행하며 아직 수행하지 않은
과업을 자동으로 하나씩 배정한다. 각 참여자는 배정된 과업을 서비스에서 수행한 직후
운영자와 함께 익명 평가를 입력한다. 각 응답은 즉시 JSONL과 중간 보고서에 저장되므로
도중에 종료돼도 앞선 응답은 유지된다. 이름, 전화번호, 이메일, 상세 건강정보는 받지
않는다.

완료 조건에는 실제 참여자 5명, 고령자 1명 이상, 보호자 1명 이상, 과업 5종 전수가
모두 포함된다. 따라서 참여자 유형 입력 시 최소 한 명은 `senior`, 최소 한 명은
`caregiver`로 실제 역할에 맞게 선택한다. 전체 수집 후 다음 명령으로 완료 조건을 검증한다.

```powershell
& ".\.venv-p1\Scripts\python.exe" scripts\collect_rise_user_evaluation.py --strict
```

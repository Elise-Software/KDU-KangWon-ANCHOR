## 제1장 사업개요

### 1. 사업목표

원주시 고령자와 보호자가 생활건강, 복지제도, 의료기관·약국·상담기관 정보를
쉽게 찾도록 공식자료 기반 RAG 챗봇을 구축한다. 근거 없는 답변을 제한하고
응급·자살·중독·의료 고위험 질의에는 안전 규칙과 즉시 연락 수단을 우선한다.

| 세부목표 | 구체적 실적 |
|---|---|
| 지역 지식베이스 50건 이상 | 공식 문서 70건, 의미 청크 580건 |
| RAG 검색·답변 | BGE-M3 임베딩, FAISS, BGE reranker, 근거 인용 응답 |
| 고령자 안내 시나리오 10건 | 서로 다른 생활건강·복지 영역 10건 strict 통과 |
| 의료·복지기관 연계 | 기존 기관 2,487건 보존, 문서 연결 기관 79개 |
| 안전응답 20건 이상 | 고유 안전 판정 패턴 31개 |
| 관리자 기능 | 질의·실패·위험·피드백 조회, 필터, CSV 내보내기 |
| 최종 결과물 1식 | DGX Docker Compose 실서비스 배포 |

### 2. 사업내용

원주시 보건·복지 공식 문서를 수집해 원문 URL, 기준일, SHA-256을 보존하고 정제·중복
제거·청킹·기관 연결을 수행했다. 검색은 BAAI/bge-m3와 FAISS를 사용하고
BAAI/bge-reranker-v2-m3로 재정렬한다. 생성 모델은 OpenAI 호환 서버를 사용하며 모든
일반 답변에 출처·문서·청크 식별자를 반환한다. 서비스 UI는 고령자 가독성, 확인 질문,
전화·지도 연결과 안전 경고 카드를 제공한다.

## 제3장 사업성과

### 1. 정량적 성과

- 공식 문서 70건, 청크 580건, 수집 오류 0건
- 연결 기관 79개, 기관 연결 레코드 110건
- Retrieval Recall@5 0.96, MRR 0.865
- 인용 정확도 1.0, factual citation relevance 0.9776
- 전체 factual groundedness 0.5892
- 고령자 시나리오 10건: Recall@5 9/9, 최소 groundedness 0.6154, 인용 정확도 1.0
- 안전 양성 12건·음성 5건, pass rate·precision 1.0
- 전체 pytest 139건 중 138건 통과, 1건 skip, 실패 0건
- 첫 실응답 2.547초, 새로고침 0회

### 2. 정성적 성과

- 범용 챗봇과 달리 원주시 공식자료와 기관 마스터를 답변 근거로 연결했다.
- 진단·처방·근거 없는 복용량 확정을 제한하고 고위험 질의를 생성 전에 분기한다.
- 일반 사용자와 개발자 모델 접근을 분리하고 원본 vLLM의 일반 접근을 차단했다.
- 관리자 감사 로그는 직접 식별정보를 마스킹하고 사용자 ID를 단방향 해시로 저장한다.
- 잘못 채운 정보보다 공백·미확인·출처 충돌을 보존하는 데이터 원칙을 적용했다.

### 3. 실제 사용자 성과

실제 사용자 5명이 익명 동의 후 5개 과업을 각각 수행했다. 참여 유형은 고령자 3명,
보호자 2명이었고 과업 5종 전수, 완료율 100%로 실제 사용자 평가 완료 기준을 충족했다.
평균 소요시간은 1,336.6초였으며 가독성 4.2/5, 명확성 3.6/5, 신뢰도 3.6/5,
기관연결 편의성 4.0/5, 안전 안내 명확성 3.8/5로 집계됐다.

주요 개선 의견은 의료 안내 표현의 명확화, 병원 운영시간·부가정보 보강, 현재 위치
자동 감지, 모바일 전화번호 바로 누르기, 영문 의료자료의 한국어 설명 보강이다. 표본이
5명으로 작고 평균 과업 시간이 약 22분 17초여서 대규모 만족도 일반화보다는 다음 UI·
데이터 개선 우선순위를 정하는 근거로 사용한다.

20대 남성 2개, 70대 여성 1개 프로필을 포함한 마스킹 합성 모집단 5개로 UI
사전점검을 수행했다. 고령자와 보호자 유형을 포함하고 실제 사용자 평가용 5개 과업을
각 1회씩 전수 수행하도록 구성했다. 모집단 수, 필수 유형, 과업 범위에 대한 합성
사전평가 내부 기준은 충족했다. 완료율은 100%, 평균 소요시간은 118.2초였고 가독성·
명확성·신뢰도는 각각 4.2/5, 기관연결 편의성과 안전 안내는 각각 4.4/5였다. 이는 실제
사람의 의견이 아니므로 실제 사용자 실적이나 외부 참여 증빙에는 포함하지 않는다.

자동 페르소나와 모의 프로필 결과는 위 실제 사용자 집계와 분리했다. 실제 사용자 원본은
직접 식별정보 없이 UUID로 저장했으며 상태는 `completed`다.

### 4. 연계성과

원주시 공식 보건·복지 문서, 지역 의료기관·약국·보건기관 데이터를 하나의 검색·기관
연결 구조로 통합했다. 지역별 공식자료와 기관 마스터를 교체할 수 있어 다른 지자체로
확장 가능한 구조다.

## 제4장 활용방안 및 기대효과

### 1. 활용방안

- 원주시 주민의 생활건강·복지·기관 탐색 서비스
- 보건·복지·요양·돌봄기관 홈페이지 또는 안내 단말 연계
- 공식자료 갱신 감시와 기관 운영정보 품질관리 도구
- 지자체별 독립 데이터·권한을 사용하는 다지역 안내 서비스

### 2. 기대효과

확정되지 않은 매출·고용·방문자 증가율은 임의로 작성하지 않는다. 실서비스 운영 후
월간 성공 질의, 출처 클릭, 전화·지도 연결, 안전 연락 연결, 사용자 만족도와 공식자료
갱신 소요시간을 기준으로 정량화한다.

### 3. 지원효과 요약

공식자료 탐색, 문서 검색, 기관 연결, 안전 라우팅, 관리자 품질관리를 단일 서비스로
구현해 지역 건강·복지 안내 서비스의 구축·검증 시간을 줄일 기반을 마련했다.

# 기업지원협력 사업계획 대비 개발 결과 평가

최종 결과보고 형식은 `R,D 및 비R,D 산학 공동 기업협력과제 지역연계 사업 결과보고서(양식)_2026. 6. 기준 (3).pdf`를 기준으로 한다. 문서 SHA-256은 `3D45765783D06C160B87778AE37C457CF91A84FEC74A72BF45B75E94984D892E`이며 15쪽이다. 계획대비 실적의 기술 목표는 최종 사업계획서의 7개 기능지표를 참조하고, 저장소·자동 테스트·`http://192.168.100.58` 실서비스를 근거로 평가했다.

## 최종 보고 범위

최종 양식은 사업개요, 계획대비 추진경과·실적, 정량·정성 성과, 활용방안·기대효과, 결과물과 부록을 요구한다. 사용자 지정 범위에 따라 경동대학교 수행 항목, SW저작권, 기자재 비교견적은 이 기술 완료 평가에서 제외한다. 실제 사용자 평가와 공동 적용성 검토는 포함하되, 자동 페르소나를 실제 사용자로 계산하거나 확인 서명을 임의 생성하지 않는다.

## 결론

개발 방향과 핵심 구조는 문서와 일치하며, RAG·지역 기관 연계·안전응답·웹 챗봇은 계획보다 구체적으로 구현됐다. 관리자용 질의·실패·피드백 감사 저장소와 운영 화면까지 DGX 실서비스에서 검증해 **기능 평가지표 7개를 모두 통과**했다. 실제 사용자 5명의 익명 평가도 완료했으며 공동 적용성은 개선 권고 조건으로 적합 판정했다. 다만 검토자 확인 서명처럼 담당자 확인이 필요한 증빙은 임의 작성하지 않는다.

## 실제 품질 평가

- 공식 문서: 70건, 수집 오류 0건
- 검색 인덱스: BAAI/bge-m3, 580 벡터, 1,024차원, FAISS cosine
- 재정렬: BAAI/bge-reranker-v2-m3
- 평가 세트: factual 100건, 안전 양성 12건, 안전 음성 5건
- Retrieval Recall@5: 0.96
- MRR: 0.865
- 사실 답변 groundedness: 0.5892
- factual citation relevance: 0.9776
- citation accuracy: 1.0
- 안전 pass rate·negative pass rate·precision: 모두 1.0
- 관리자 API·정량 증빙 테스트: `43 passed`
- 관리자 실브라우저 테스트: `16 passed`
- 배포 테스트 수: `74개`

- 실서비스 `/health`, `/api/config`: HTTP 200
- 실서비스 감사 권한: 일반 사용자 조회 403, 피드백 200, 관리자 조회·CSV 200

groundedness는 설정 임계치 0.55를 넘지만 0.5892로 여유가 크지 않다. 문서 목표에는 수치 기준이 없어서 통과이나, 대외 시연 품질을 높이려면 가장 먼저 개선할 지표다.

## 개발 내용 일치 여부

일치하거나 초과 구현된 내용:

- 웹 기반 챗봇 UI와 관리자 페이지
- 공공 건강·복지 문서 수집 및 지역 기관·약국 연결
- RAG 검색, 벡터 인덱스, reranker, 근거 URL·청크 인용
- 진단·처방 제한, 응급·자살·중독 등 고위험 안전 라우팅
- 고령자 가독성, 확인 질문, 전화·지도 연결
- Docker Compose 및 DGX 실배포

실제 사용자 평가 결과:

- 실제 참여자 5명: 고령자 3명, 보호자 2명
- 과업 5종 전수, 완료율 100%, 상태 `completed`
- 가독성 4.2, 명확성 3.6, 신뢰도 3.6, 기관연결 4.0, 안전 안내 3.8
- 공동 적용성: 적합(개선 권고), 검토자 확인 서명은 별도 필요

마스킹 합성 모집단 5개를 사용한 사전평가는 고령자·보호자 유형과 5개 과업 전수를
포함해 내부 기준을 통과했다. 이 결과는 기능 적용 범위 점검에는 사용하지만 실제 참여자
성과나 공동 확인을 대체하지 않는다.

## RISE 선정평가 배점표에 따른 잠정 자기평가

이 점수는 외부위원의 공식 점수가 아니라 현재 저장소에서 확인되는 기술 증거만으로 산정한 잠정치다.

| 평가항목 | 배점 | 잠정점수 | 근거 |
|---|---:|---:|---|
| 지역수요 기반 창의성·혁신성 | 15 | 14 | 원주시 고령자·기관 데이터와 안전 RAG를 결합 |
| 연구계획 충실성 | 15 | 14 | 계획의 4단계 구조와 실제 구현이 정합하며 시험·배포까지 완료 |
| 과제 수행 연구역량 | 25 | 21 | 완성된 시스템은 강한 증거이나 인력·연구실적 자료 부족 |
| 파급효과·시장성·활용가능성 | 20 | 17 | 실제 지역 서비스와 확장 가능한 배포 구조 확보 |
| 성과창출전략·지식재산권·산업화 | 25 | 14 | 실서비스·문서는 있으나 등록·로드맵·논문 증빙 부족 |
| **합계** | **100** | **80** | 문서상 원칙적 선정 기준 70점 이상 |

80/100은 사업 선정평가 배점표에 대한 과거 잠정 자기평가이며 최종 결과보고서 양식에는 별도의 100점 배점이 없다. 최종보고 관점에서는 기능개발 7/7, 정량 기술성과와 실제 사용자 평가가 완료됐고 공동 적용성은 개선 권고 조건으로 적합하다.
